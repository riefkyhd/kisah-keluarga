# User Roles and Permissions

## Role Overview

### 1. Viewer
Can:
- login
- browse family directory
- view member profiles
- view family tree
- view stories / memories that are published

Cannot:
- edit member data
- delete members
- manage permissions

### 2. Contributor
Can:
- do everything viewer can
- submit edits / suggestions
- upload profile photo suggestions
- add story / memory drafts

Cannot:
- directly publish changes to critical relationship data unless approved
- manage roles

### 3. Editor
Can:
- create members
- edit member profile fields
- create or remove relationships
- upload / replace profile photos
- archive members
- publish stories and memories

Cannot:
- manage system settings deeply
- assign admin role

### 4. Admin
Can:
- do everything editor can
- manage roles
- resolve duplicates
- review audit logs
- manage privacy settings
- configure branch visibility and moderation rules

## Permission Philosophy
- Most family members should be **viewers**.
- A smaller trusted group should be **editors**.
- Only one or two people should be **admins**.
- Contributors are useful if the family wants crowdsourced data with review.

## Recommended Initial Setup
- 1 super admin
- 1–3 editors
- everyone else viewer
