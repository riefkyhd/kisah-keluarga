# Non-Functional Requirements

## Performance
- directory search should feel responsive
- profile pages should load quickly on mobile
- large image uploads should fail gracefully with guidance

## Reliability
- app should not depend on fragile client-only logic for critical operations
- deployment should be predictable
- error states should be recoverable

## Maintainability
- keep architecture understandable by future maintainers
- avoid unnecessary abstraction early
- document major decisions

## Accessibility
- keyboard-friendly where applicable
- readable text sizes
- clear labels and forms
- good color contrast

## Scalability
The app should comfortably scale from a small family to a large extended family without redesigning the core data model.
