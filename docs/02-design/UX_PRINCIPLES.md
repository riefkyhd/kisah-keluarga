# UX Principles

## UX North Star
Aplikasi harus terasa seperti **buku keluarga digital yang sederhana**, bukan dashboard enterprise dan bukan software genealogy rumit.

## Main UX Principles

### 1. People First, Diagram Second
Halaman utama harus memudahkan user mencari dan membuka profil anggota keluarga.
Tree/graph view adalah fitur penting, tapi bukan entry point utama.

### 2. One Main Action Per Screen
Setiap layar harus punya aksi utama yang jelas.
Contoh:
- halaman direktori -> cari orang
- halaman profil -> lihat hubungan / edit sederhana
- halaman edit -> simpan perubahan

### 3. Editing Must Feel Local and Natural
Menambah anak sebaiknya dari profil orang tua.
Menambah pasangan sebaiknya dari profil orang yang bersangkutan.
Jangan memaksa user membuka form besar dan umum.

### 4. Reduce Cognitive Load
- gunakan label sederhana
- hindari istilah teknis berlebihan
- gunakan foto profil dan hubungan yang jelas
- gunakan empty states yang membantu

### 5. Mobile First
Mayoritas keluarga kemungkinan akan membuka dari HP.
Semua flow inti wajib nyaman di mobile:
- login
- search
- buka profil
- tambah anggota
- upload foto
- menghubungkan relasi

### 6. Safety Over Speed for Destructive Actions
Penghapusan harus default ke archive / soft delete.
Jangan ada destructive action tanpa konfirmasi yang sangat jelas.
