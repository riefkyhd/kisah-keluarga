# Information Architecture

## Main Navigation

### Primary Navigation
- Beranda
- Anggota Keluarga
- Cabang Keluarga
- Cerita Keluarga
- Timeline
- Admin (role-based)

## Main Screens

### 1. Home
Purpose:
- welcome screen
- search entry
- recent updates
- highlighted family stories
- shortcuts to major sections

### 2. Family Directory
Purpose:
- browse and search all members
- filter by branch / generation

### 3. Member Profile
Purpose:
- become the main information hub for one person

Contains:
- photo
- basic details
- parents
- spouse
- children
- siblings (derived)
- branch
- memories / stories snippet
- edit actions (if allowed)

### 4. Tree View
Purpose:
- visualize relationships around one person
- should not replace profile view

### 5. Stories / Memories
Purpose:
- keep family app emotionally rich
- allow stories, old photos, and notes

### 6. Admin Area
Purpose:
- role management
- moderation
- audit logs
- duplicates
- archived members

## Content Hierarchy
1. person
2. relationships
3. stories/memories
4. media assets
5. branches
6. events/timeline
