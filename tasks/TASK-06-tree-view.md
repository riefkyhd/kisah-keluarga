# TASK-06 Tree View

## Objective
Introduce a simple and understandable tree view.

## Deliverables
- focused graph view for one member
- parent / spouse / child visualization
- pan/zoom
- reset view control
- fallback relation list remains available

## Acceptance Criteria
- users can visually understand direct family relationships
- tree does not replace the simpler profile-based view

## Important Note
Do not overbuild this phase before the CRUD and relationship model are stable.
