# TASK-02 Family Members CRUD

## Objective
Build the main people directory and member create/edit/archive flow.

## Deliverables
- `people` table migration
- family directory page
- member profile page
- create member form
- edit member form
- archive / restore actions

## Acceptance Criteria
- editor/admin can create member
- editor/admin can edit member
- archived members do not show in normal list
- member profile loads correctly

## UX Notes
- forms should stay simple
- default required fields should be minimal
- mobile experience must be comfortable
