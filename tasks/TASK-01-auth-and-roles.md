# TASK-01 Auth and Roles

## Objective
Implement authentication and role system.

## Deliverables
- login page
- auth callback flow
- user role table
- role guard utilities
- protected admin routes

## Acceptance Criteria
- user can log in
- protected routes block unauthorized users
- admin/editor/viewer checks work server-side

## Important Notes
- keep auth flow simple
- prepare for email magic link / OTP style flow if preferred
- do not trust client-only role guards
